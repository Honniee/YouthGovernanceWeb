## 1. Purpose and Scope

This Data Processing Agreement (DPA) outlines the procedures and responsibilities for processing personal data collected through the Youth Governance Web platform in compliance with the Data Privacy Act of 2012 (RA 10173).

## 2. Legal Basis

Data processing is conducted under the following legal authorities:

1. **Republic Act No. 8044** (Youth in Nation-Building Act)
   - Mandates local government units to establish youth development programs
   - Requires collection of youth data for program planning and implementation

2. **Republic Act No. 10173** (Data Privacy Act of 2012)
   - Provides framework for personal data protection
   - Establishes rights of data subjects
   - Sets requirements for data controllers and processors

3. **Local Government Code of 1991**
   - Grants local government units authority to collect data for public service delivery

### 4.2 Contact Information

## 5. Purpose of Data Processing

Personal data is processed for the following purposes:

1. **Youth Program Planning**
   - Analyze youth demographics and needs
   - Design targeted youth programs and services
   - Allocate resources effectively

2. **Survey Administration**
   - Conduct KK Survey and other assessments
   - Validate survey responses
   - Generate statistical insights

3. **Service Delivery**
   - Manage youth profiles and accounts
   - Facilitate program registration
   - Communicate with youth participants

4. **Legal Compliance**
   - Comply with legal obligations
   - Respond to government requests
   - Maintain audit trails

5. **Research and Statistics**
   - Generate statistical reports
   - Conduct research on youth issues
   - Support policy development

## 6. Data Processing Procedures

### 6.1 Data Collection
- Data is collected directly from data subjects through:
  - Online registration forms
  - Survey questionnaires
  - Program applications
  - Contact forms

### 6.2 Data Storage
- Data is stored in secure PostgreSQL database
- Encryption in transit (SSL/TLS) and at rest
- Regular automated backups
- Access controls and authentication

### 6.3 Data Processing
- Automated processing for:
  - Profile management
  - Survey validation
  - Statistical analysis
  - Report generation
- Manual processing for:
  - Data validation
  - Quality assurance
  - Support requests

### 6.4 Data Sharing
- Data is shared only with:
  - Authorized LYDO staff (role-based access)
  - SK officials (for validation purposes)
  - Service providers (under strict agreements)
  - Government authorities (when legally required)

### 6.5 Data Retention
- Youth profiles: 5 years after last activity
- Survey responses: 5 years after creation
- Activity logs: 90 days
- Audit logs: 1 year

### 6.6 Data Disposal
- After retention period:
  - Personal data is anonymized (identifiers removed)
  - Aggregated data retained for statistics
  - Original data permanently deleted

## 7. Security Measures

### 7.1 Technical Measures
- **Encryption:** SSL/TLS for data in transit, encryption at rest
- **Access Controls:** Role-based access control (RBAC)
- **Authentication:** Strong password requirements, JWT tokens
- **Backups:** Automated daily backups with retention policies
- **Monitoring:** Security monitoring and incident detection
- **Updates:** Regular security updates and patches

### 7.2 Organizational Measures
- **Staff Training:** Regular training on data privacy and security
- **Access Management:** Principle of least privilege
- **Audit Logs:** Comprehensive logging of data access and modifications
- **Incident Response:** Security incident response plan
- **Data Protection Officer:** Designated DPO for privacy matters

### 7.3 Physical Measures
- **Server Security:** Secure data center facilities
- **Access Controls:** Physical access restrictions
- **Disaster Recovery:** Backup and recovery procedures

## 8. Data Subject Rights

Data subjects have the following rights:

1. **Right to be Informed** - Transparency about data processing
2. **Right to Access** - Request access to personal data
3. **Right to Rectification** - Request correction of inaccurate data
4. **Right to Erasure** - Request deletion of personal data
5. **Right to Object** - Object to data processing
6. **Right to Data Portability** - Request data in portable format
7. **Right to File a Complaint** - File complaint with NPC

## 9. Data Breach Procedures

In the event of a data breach:

1. **Detection and Assessment**
   - Immediate detection and assessment of breach
   - Determine scope and impact
   - Classify severity (low/medium/high/critical)

2. **Containment**
   - Immediate containment of breach
   - Prevent further unauthorized access
   - Preserve evidence for investigation

3. **Notification**
   - Notify National Privacy Commission within 72 hours
   - Notify affected data subjects as soon as practicable
   - Notify relevant authorities if required

4. **Remediation**
   - Investigate root cause
   - Implement corrective measures
   - Restore system security

5. **Post-Incident Review**
   - Conduct post-incident review
   - Update security measures
   - Document lessons learned

## 10. Third-Party Processors

When using third-party processors:

1. **Due Diligence** - Assess processor's security and privacy practices
2. **Contracts** - Execute data processing agreements
3. **Monitoring** - Monitor processor's compliance
4. **Audit Rights** - Retain right to audit processors
